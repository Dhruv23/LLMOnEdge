Predicting GPT-2 Inference Latency

Noise vs. Signal on Edge Hardware

Dhruv | Feb 2026

The Challenge: Raw Execution Time

  • High Variance: Scatter plot shows massive jitter. At 50 tokens, latency fluctuates between 2.4ms and 4.0ms.
  • Model Failure: $R^2 \approx 0.10$. Context length explains only 10% of variance.
  • Cause: 90% of performance delta is system noise (clock speed, memory contention), not compute load.

The Solution: Median Behavior

  • Noise Filtering: Aggregating to Medians reveals the true linear trend.
  • High Accuracy: $R^2$ jumps to ~0.99.
  • Insight: Random Forest (Blue) captures the "step" increases caused by memory allocation blocks, which Linear Regression (Red) misses.

Conclusion & Recommendations

  • System Jitter Dominates: You cannot guarantee exact latency for a single run due to hardware noise.
  • Averages are Predictable: Capacity planning should rely on the Median model, which is highly accurate.
  • Next Steps: To fix the noise seen in Slide 2, investigate GPU Clock Locking or Memory Pinning.
