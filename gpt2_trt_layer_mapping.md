# GPT-2 Medium TensorRT Layer Mapping

This document maps the theoretical macro stages of the GPT-2 Medium architecture to their standard Hugging Face PyTorch layer names, and lists their typical equivalent fused operations when compiled into a TensorRT engine.

This is a **mock map** based on expected TensorRT Turing architecture fusions. The actual fused names may vary based on TensorRT version and precision (`--fp16`). You can update the `Expected TensorRT Fused Layer Name` column using the JSON generated by `dump_engine_info.sh`.

## Architecture Overview (GPT-2 Medium)
- **Parameters:** ~345M
- **Vocabulary Size:** 50,257
- **Hidden Size:** 1024
- **Layers:** 24
- **Attention Heads:** 16

## Notes on TensorRT Fusions
- **Fused Attention (MHA/FMHA):** TensorRT uses highly optimized kernels (like FlashAttention or specific Turing/Ampere fused kernels) that condense the `Q*K^T -> Softmax -> *V` steps into a single `fused_attention` node.
- **LayerNorm + Residual Add:** TensorRT aggressively fuses `Add` (residual connection) with the subsequent `LayerNorm`. Expect to see layers named similar to `fused_add_layernorm`.
- **GELU:** The Gaussian Error Linear Unit activation is almost always fused with the preceding Fully Connected (FC) layer or generated as a specialized point-wise `pw_gelu` operation.
- **QKV Matrices:** GPT-2 uses a single `c_attn` weight matrix for Q, K, and V. TensorRT typically maintains this as a single large MatMul and splits the result, often labeled as `fused_qkv`.
